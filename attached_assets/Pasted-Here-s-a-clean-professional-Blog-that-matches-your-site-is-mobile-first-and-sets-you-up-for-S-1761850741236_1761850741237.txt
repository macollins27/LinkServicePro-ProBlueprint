## 2) Content model (lightweight, easy to migrate)

For now, store posts in `src/data/blog-posts.ts` as an array. Each item:

```ts
export type BlogPost = {
  slug: string;
  title: string;
  excerpt: string;
  html?: string;           // optional pre-rendered HTML
  content?: string;        // optional markdown string (render later)
  category: 'Operations' | 'Sales' | 'Product' | 'Finance' | 'People' | string;
  tags: string[];
  author: {
    name: string;
    role?: string;
    avatarUrl?: string;
  };
  cover?: { alt: string };
  publishedAt: string;     // ISO
  updatedAt?: string;      // ISO
  readMinutes: number;
};
```

You can later switch to Markdown/MDX files or CMS; routes stay the same.
